# Chatroom 聊天室 — 需求文档

## 1. 项目概述

构建一个类QQ的实时聊天室应用，支持用户注册登录、好友管理、一对一私聊、群组聊天等核心功能。系统需具备处理高并发的能力，确保消息的即时性、一致性和可靠性。

## 2. 功能需求

### 2.1 用户模块
- **注册**: 用户通过用户名+密码注册，密码BCrypt加密存储
- **登录**: JWT Token认证，Token有效期7天
- **个人信息**: 查看/修改昵称、头像
- **在线状态**: 实时显示用户在线/离线状态

### 2.2 好友模块
- **添加好友**: 通过用户名/昵称搜索用户并发送好友申请
- **处理申请**: 接受/拒绝好友申请
- **删除好友**: 解除好友关系
- **好友列表**: 展示所有好友及其在线状态

### 2.3 一对一聊天
- **实时消息**: WebSocket推送，消息即时送达
- **消息历史**: 分页拉取历史消息，保留30天
- **消息引用回复**: 支持引用某条消息进行回复
- **消息撤回**: 2分钟内支持撤回已发送消息
- **已读状态**: 消息送达/已读状态追踪

### 2.4 群组聊天
- **创建群组**: 选择好友创建群聊
- **群成员管理**: 邀请/移除成员（群主/管理员权限）
- **退出群组**: 成员可选择退出
- **群组消息**: 群内广播消息，所有成员实时接收

### 2.5 扩展预留
- 消息数据保留30天，支持后续蒸馏训练机器人自动回复
- 预留消息导出接口

## 3. 非功能需求

### 3.1 性能要求
- WebSocket支持1000+并发连接
- 消息发送到送达延迟 < 500ms
- API响应时间 < 200ms (P95)
- 支持水平扩展（无状态服务设计）

### 3.2 可靠性要求
- 消息不丢失: 先推送后持久化 + 客户端ACK确认
- 消息有序: 全局自增消息ID保证顺序
- 消息去重: 客户端生成唯一ID防止重复发送

### 3.3 安全性要求
- 密码BCrypt加密
- JWT Token认证
- API接口权限控制
- WebSocket连接Token验证

### 3.4 数据保留
- 聊天记录保留30天
- 每日定时清理30天前的消息
- 保留策略可配置

## 4. 技术约束
- 后端: Spring Boot 3.2 + Java 17
- 前端: Vue 3 + Element Plus
- 数据库: MySQL 8.0 (Demo使用H2)
- 通信: WebSocket (STOMP协议) + REST API
- 缓存: Redis (生产环境)
- 消息队列: RabbitMQ (生产环境)
